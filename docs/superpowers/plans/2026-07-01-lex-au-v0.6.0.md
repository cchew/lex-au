# lex-au v0.6.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Two corpus-quality improvements — run-level inline formatting (`<b>/<i>/<sup>/<sub>`) from DOCX runs, and term/def injection for list-form definitions (`X means:` followed by block-list items).

**Architecture:** Inline formatting stores run-level spans in `ParsedParagraph`, reads them in `docx_reader`, and emits `<b>/<i>/<sup>/<sub>` children into `<p>` elements in `builder`. List-form definitions are detected in a new post-processing pass `inject_list_defs` in `termlinks.py` that converts `<content>` to `<intro>` and injects `<term>` into the lead `<p>`. Both features require `inject_terms._process_p` to handle mixed-content `<p>` first (v0.6.0 starts with that guard).

**Tech Stack:** Python 3.12, lxml, python-docx (runs via `paragraph.runs`), pytest. No new dependencies.

## Global Constraints

- Python ≥ 3.12, all type annotations required.
- AKN namespace: `"http://docs.oasis-open.org/legaldocml/ns/akn/3.0"` — never hard-code the URI string inline, always use the `AKN_NS` / `AKN_TAG` constants already defined in each module.
- Tests live in `tests/`, one file per source module. Run with `pytest` from the repo root (venv: `source .venv/bin/activate`).
- Existing 231 tests must pass at every commit.
- All `ParsedParagraph` changes must be backwards-compatible: callers that construct `ParsedParagraph(ElementType.SECTION, number="1", heading="Foo")` must continue to work without providing `spans`.
- `_emit_p_inline` must be a module-level function in `builder.py` (not a method), so `_build_preface` and `_build_schedule_content` can call it without going through `AknBuilder`.
- `<tocItem>`, `<num>`, `<heading>` element text is structural — never apply inline formatting to them, only to `<p>` elements.
- Schedule clause heading text extracted by regex groups is also plain — leave those `etree.SubElement(...).text = heading_str` lines unchanged.
- Version bump: `pyproject.toml` `version = "0.5.2"` → `"0.6.0"`. README version table updated.

```python
def iter_paragraphs(doc: Document) -> Iterator[ParsedParagraph]:
    """Yield ParsedParagraph for each Paragraph and Table in document order.

    Tables are yielded as ParsedParagraph(TABLE, table_rows=[[cell, ...], ...]).
    FIGURE paragraphs (inline image) yield with empty spans.
    All other paragraphs populate spans from paragraph.runs.
    """
    for block in doc.iter_inner_content():
        if isinstance(block, Paragraph):
            if _has_inline_image(block):
                yield ParsedParagraph(ElementType.FIGURE, text=block.text)
                continue
            style = block.style.name if block.style else "Default"
            spans = [
                InlineSpan(
                    text=run.text,
                    bold=bool(run.bold),
                    italic=bool(run.italic),
                    superscript=bool(run.font.superscript),
                    subscript=bool(run.font.subscript),
                )
                for run in block.runs
                if run.text
            ]
            full_text = "".join(s.text for s in spans)
            parsed = parse_paragraph(style, full_text)
            level = _list_level(block)
            if level is not None and parsed.element_type == ElementType.BODY:
                from dataclasses import replace as _replace
                parsed = _replace(parsed, element_type=ElementType.LIST_ITEM, number=str(level))
            # Attach spans (use full span list even when classification used full_text)
            from dataclasses import replace as _replace
            parsed = _replace(parsed, spans=spans)
            yield parsed
        elif isinstance(block, Table):
            rows = [
                [cell.text.strip() for cell in row.cells]
                for row in block.rows
            ]
            yield ParsedParagraph(element_type=ElementType.TABLE, table_rows=rows)
```

## Task 3: Update `inject_terms._process_p` for mixed-content `<p>`

**Files:**
- Modify: `src/lexau/termlinks.py`
- Test: `tests/test_termlinks.py`

**Why first:** `builder.py` (Task 4) will start emitting `<b>/<i>` children into `<p>` elements. `inject_terms` currently skips any `<p>` with children. Without this fix, inline-formatted definition paragraphs would lose their term/def markup. This task makes `inject_terms` robust to mixed content before Task 4 introduces it.

**Approach:** When `<p>` has children, extract text via `itertext()`, run pattern matching on that, then clear the element and rebuild with `<term>/<def>` (losing inline markup, which is acceptable — `<term>` carries the semantic weight that the italic was implying).

```python
def _process_p(
    p_el: etree._Element,
    registry: dict[str, str],
) -> int:
    """Try to inject <term> + <def> into a single <p>. Returns 1 if injected, 0 otherwise.

    Handles both plain-text <p> and mixed-content <p> (e.g. with <i>/<b> children from
    inline formatting). For mixed content, text is reconstructed via itertext(); on match,
    inline children are cleared before rebuilding with <term>/<def>.
    """
    children = list(p_el)
    if children:
        text = "".join(p_el.itertext()).strip()
    else:
        text = p_el.text
    if not text:
        return 0

    if _STOP_DEFINIENDUM.match(text.strip()):
        return 0

    for pattern in _DEF_PATTERNS:
        m = pattern.match(text.strip())
        if m:
            show_as = m.group(1).strip()
            connector = m.group(2).strip()
            definiens = m.group(3).strip()
            eid = _term_eid(show_as)
            registry[eid] = show_as

            original = m.group(0)
            quoted = original.startswith('"')

            # Clear element (works for both plain text and mixed content)
            p_el.text = None
            for child in list(p_el):
                p_el.remove(child)

            term_el = etree.SubElement(p_el, f"{AKN_TAG}term")
            term_el.set("refersTo", f"#{eid}")
            term_el.text = f'"{show_as}"' if quoted else show_as
            term_el.tail = f" {connector} "
            def_el = etree.SubElement(p_el, f"{AKN_TAG}def")
            def_el.text = definiens
            return 1

    return 0
```

```python
def _emit_p_inline(p_el: etree._Element, p: "ParsedParagraph") -> None:
    """Emit p.text or inline children into p_el, preserving run-level formatting.

    If p.spans is empty or all spans are unformatted, falls back to p_el.text = p.text
    (same behaviour as before v0.6.0). Otherwise emits <b>, <i>, <sup>, <sub> children.
    Bold+italic is rendered as <b><i>text</i></b>.
    """
    from lexau.parser import InlineSpan  # avoid circular at module level
    if not p.spans or not any(
        s.bold or s.italic or s.superscript or s.subscript for s in p.spans
    ):
        p_el.text = p.text
        return

    prev: etree._Element | None = None
    for span in p.spans:
        if not span.text:
            continue
        if span.bold or span.italic or span.superscript or span.subscript:
            if span.bold and span.italic:
                outer = etree.SubElement(p_el, f"{{{AKN_NS}}}b")
                child = etree.SubElement(outer, f"{{{AKN_NS}}}i")
                child.text = span.text
                prev = outer
            elif span.bold:
                child = etree.SubElement(p_el, f"{{{AKN_NS}}}b")
                child.text = span.text
                prev = child
            elif span.italic:
                child = etree.SubElement(p_el, f"{{{AKN_NS}}}i")
                child.text = span.text
                prev = child
            elif span.superscript:
                child = etree.SubElement(p_el, f"{{{AKN_NS}}}sup")
                child.text = span.text
                prev = child
            else:  # subscript
                child = etree.SubElement(p_el, f"{{{AKN_NS}}}sub")
                child.text = span.text
                prev = child
        else:
            # Plain span — append as tail of last element, or text of p_el
            if prev is None:
                p_el.text = (p_el.text or "") + span.text
            else:
                prev.tail = (prev.tail or "") + span.text
```

## Task 5: `inject_list_defs` for X means: list-form definitions

**Files:**
- Modify: `src/lexau/termlinks.py`
- Test: `tests/test_termlinks.py`

**Problem:** `sensitive information means:` (and 32+ others in Privacy Act; 148+ in Social Security Act) currently emits as `<content><p>sensitive information means:</p></content>` with no `<term>` markup. The definition body is in following `<paragraph>` siblings.

**AKN solution:** Convert `<content>` to `<intro>` (which introduces a block list in AKN 3.0) and inject `<term>` into the `<p>`. This is valid AKN — `<intro>` is designed to introduce following block children.

**Detection logic:**
1. Walk all `<section>` elements that `_is_definition_section` returns True for.
2. For each `<content>` element with exactly one `<p>` child whose full text matches `X means:` or `X means (qualifier):` (text ends with `means:` or `means (...):`)
3. Confirm the parent element has at least one following sibling `<paragraph>` or `<blockList>` element.
4. Convert `<content>` tag to `<intro>`, inject `<term>` into the `<p>`.

**Pattern:** `r'^(.+?)\s+means\s*:$'` with `re.IGNORECASE` — captures everything before ` means:` as the definiendum. Guard with `_STOP_DEFINIENDUM` to avoid "This Act means:".

**Term text extraction:** Use the full text before ` means:` as `show_as`. For terms with qualifiers like "contracted service provider, for a government contract, means:", take only the text before the first comma: `show_as = m.group(1).split(",")[0].strip()`. This gives "contracted service provider".

```python
def inject_list_defs(
    root: etree._Element,
    registry: dict[str, str],
) -> int:
    """Detect 'X means:' list-form definitions and inject <term> + convert <content> to <intro>.

    Processes only definition sections (same gate as inject_terms).
    Mutates registry in place. Returns count of injected terms.
    """
    count = 0

    for section_el in root.iter(_SECTION_TAG):
        if not _is_definition_section(section_el):
            continue

        # Walk all <content> elements inside this section
        for content_el in list(section_el.iter(_CONTENT_TAG)):
            # Must have exactly one <p> child
            p_children = list(content_el)
            if len(p_children) != 1 or p_children[0].tag != _P_TAG:
                continue
            p_el = p_children[0]

            # Get text (handles plain or mixed content from inline formatting)
            if list(p_el):
                text = "".join(p_el.itertext()).strip()
            else:
                text = (p_el.text or "").strip()

            if not text:
                continue
            if _STOP_DEFINIENDUM.match(text):
                continue

            m = _LIST_DEF_COLON_RE.match(text)
            if not m:
                continue

            # Check parent has at least one following <paragraph> or <blockList> sibling
            parent = content_el.getparent()
            if parent is None:
                continue
            siblings = list(parent)
            try:
                idx = siblings.index(content_el)
            except ValueError:
                continue
            following = siblings[idx + 1:]
            if not any(c.tag in {_PARA_TAG, _BLOCKLIST_TAG} for c in following):
                continue

            # Extract definiendum — trim at first comma to strip qualifiers
            raw_definiendum = m.group(1).strip()
            show_as = raw_definiendum.split(",")[0].strip()
            eid = _term_eid(show_as)
            registry[eid] = show_as

            # Rename <content> → <intro>
            content_el.tag = _INTRO_TAG

            # Rebuild <p>: inject <term> + keep " means:" as tail
            p_el.text = None
            for child in list(p_el):
                p_el.remove(child)
            term_el = etree.SubElement(p_el, f"{AKN_TAG}term")
            term_el.set("refersTo", f"#{eid}")
            term_el.text = show_as
            # Reconstruct tail: everything after the definiendum in original text
            tail = text[len(raw_definiendum):]  # e.g. " means:" or ", qualifier, means:"
            # Simplify tail to just " means:"
            term_el.tail = " means:"

            count += 1

    return count
```
